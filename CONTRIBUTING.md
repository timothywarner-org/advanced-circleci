# Contributing

Thank you for your interest in improving this project! Contributions of all sizes are welcome. Please review the guidelines below before opening an issue or pull request.

## Ways to Contribute
- Report bugs or request enhancements by opening an issue.
- Improve course materials or examples to better illustrate CircleCI concepts.
- Add documentation for workflows, configuration patterns, or orb integrations.

## Ground Rules
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md).
- Use clear, reproducible steps when reporting issues.
- Keep pull requests focused. Separate unrelated changes into different PRs.
- Write descriptive commit messages and PR summaries.

## Development Process
1. Fork the repository and create a feature branch.
2. Make your changes with appropriate tests or validation steps when possible.
3. Update documentation when behavior or guidance changes.
4. Open a pull request that describes the change, impact, and testing performed.
