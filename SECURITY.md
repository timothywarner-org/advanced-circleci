# Security Policy

We take security seriously and appreciate your help in responsibly disclosing vulnerabilities.

## Reporting a Vulnerability

- Email security concerns to Tim Warner at <tim@techtrainertim.com>.
- Include a description of the issue, affected files or configurations, and steps to reproduce if possible.
- Please do not create public issues for security reports until the vulnerability has been reviewed and addressed.

## Supported Versions

This project is provided as reference material. Security updates are issued on a best-effort basis; please ensure you validate any configurations before using them in production pipelines.

## Preferred Languages

Communications in English are preferred to ensure quick handling of the report.
